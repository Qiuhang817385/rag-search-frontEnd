# 主题（明暗）和 Streamdown 代码块：小白指南

这篇文档写给：**刚开始做前端、不太熟悉 Tailwind / Shiki / Streamdown** 的同学。  
目标是让你知道：**页面为什么会「有的地方黑了、有的地方没黑」**，以及**以后要改主题该动哪里**。

---

## 1. 先用生活比喻理解三件事

想象你在房间里看一本书：

| 比喻 | 技术上大致对应 | 管的是什么 |
|------|----------------|------------|
| **房间开灯还是关灯** | 整个网站的「浅色 / 深色」模式 | 背景、边框、大部分文字颜色 |
| **书页是白纸还是灰纸** | 某一个组件自己的底色（比如卡片） | 局部容器背景 |
| **正文里的彩色记号笔** | 代码语法高亮（Shiki） | `function`、`字符串` 各是什么颜色 |

这三件事**互相独立又互相叠加**。  
所以会出现：**记号笔颜色已经是「深色主题配色」，但房间灯还是亮的**——看起来就像「主题没切换」。

---

## 2. 你项目里和「明暗」相关的三类设置（按顺序记）

### ① 全站颜色变量（像「房间整体灯光」）

一般在 `app/globals.css` 里，例如 `:root { ... }`。  
这里定义 `--background`、`--foreground` 等，决定页面大基调。

### ② Tailwind 的 `dark:`（像「关灯后家具要不要换一套样式」）

很多组件会写：`浅色用白底` + `深色用灰底`。  
在 Tailwind **第 4 版**里，默认规则大致是：

- **看你电脑系统**是不是深色模式（`prefers-color-scheme`）。

这意味着：**只在某一个 `<div>` 上手动加 `class="dark"`，往往不会让 `dark:` 生效**——除非你再改 Tailwind 的配置方式（下面「进阶」会提）。

### ③ Streamdown + 代码插件（像「代码这块用什么配色高亮」）

- **Streamdown**：把 Markdown（包括代码块）渲染成网页。
- **`@streamdown/code`**：负责 **语法高亮**，底层常用 **Shiki**。
- 主题一般是 **`['浅色语法主题', '深色语法主题']`** 这一对，和「网站外壳」是不是深色不是同一个开关。

重要一点（很多人踩坑）：

> 只要你给 Streamdown 配了 **`plugins.code`**（代码插件），  
> **高亮用哪套主题，主要由这个插件决定**，  
> **而不是** `<Streamdown shikiTheme={...} />` 那一行（在当前版本逻辑里容易被插件盖住）。

所以我们后来在页面里用 **`createCodePlugin({ themes: [...] })`** 来改语法配色。

---

## 3. 你曾经遇到的「选了深色没感觉」是怎么回事？

简化成两句话：

1. **语法颜色（记号笔）**可能已经换成深色主题了（字的颜色对了）。
2. **代码框外壳**（灰框、侧边栏色）用的是 Tailwind 的 **`bg-sidebar` 一类样式**，它们要靠 **`dark:`** 或你自己的 CSS 才会变暗。

而当时的情况是：**Tailwind 默认不认你在局部加的 `dark` class**，只认**系统**深色。  
所以外壳仍偏亮，看起来像「没生效」。

后来加了 **`data-rag-code-appearance`** + 一段只作用于回复区域的 CSS，相当于：**不依赖 Tailwind 的 `dark:`，强行把这一块代码框画深**，这是「补丁」，不是最终最完美的全站方案。

---

## 4. 小白推荐你怎么选路线？

### 路线 A：不做手动切换，只跟电脑系统（最简单）

- 不改 Tailwind 默认行为。
- 尽量让 `:root` 在「系统深色」时用深色变量（你已有 `@media (prefers-color-scheme: dark)` 的例子）。
- Streamdown 用官方默认的一对主题即可：  
  **`['github-light', 'github-dark']`**。

适合：个人工具页、不需要「浅色/深色按钮」的产品。

### 路线 B：网站自己要有一个「深色 / 浅色 / 跟随系统」开关（常见产品形态）

要做两件事（概念上）：

1. **改 Tailwind**：让 `dark:` 跟 **`html` 上有没有某个 class（例如 `dark`）** 挂钩（官方文档叫 `@custom-variant dark (...)`）。
2. **写一小段脚本**：页面加载时根据用户选择（或 localStorage）决定 **`html` 要不要带 `dark`**。

做完以后：

- 全站组件的 `dark:` 会一起亮/暗；
- Streamdown 代码块外壳若用了 `dark:`，也会跟着变；
- 代码插件继续用 **`github-light` + `github-dark`** 这一对即可，不必再「两个都叫 github-dark」那种调试写法。

---

## 5. Streamdown 这边你只要记住的「最短清单」

1. **扫样式**：`globals.css` 里要有 `@source` 指向 `streamdown` 和 `@streamdown/*`，否则有些类名 Tailwind 扫不到。
2. **改语法主题**：用 **`createCodePlugin({ themes: ['github-light', 'github-dark'] })`**（或你喜欢的其它 Shiki 主题名）。
3. **`shikiTheme` prop**：挂着 `plugins.code` 时，**不要指望它单独起作用**；以插件主题为准。
4. **外壳颜色不对**：先分清是 **全站暗色没开**，还是 **只有代码块需要补丁**；长期尽量走「全站 `html.dark`」而不是到处写特例 CSS。

---

## 6. 和你当前仓库可能对应的位置（方便你对照）

| 大概是什么 | 文件（可能在） |
|------------|----------------|
| 全站全局样式、Tailwind | `app/globals.css` |
| 页面最外层 HTML | `app/layout.tsx` |
| RAG 里 Streamdown + 代码插件 | `app/ragSearch/RagChatPanel.tsx` |
| 仅 RAG 回复区的代码框深色补丁 | `globals.css` 里 `[data-rag-code-appearance='dark'] ...` 那一段 |

---

## 7. 读完还是懵：你只要记住一句

**「网站明暗」和「代码语法彩色」是两套系统；**  
**要让代码框「框」也变暗，要么跟系统走，要么让整页的 `dark` 规则真的生效，不能只靠在一个小 div 上写 `dark`（在 Tailwind v4 默认下）。**
